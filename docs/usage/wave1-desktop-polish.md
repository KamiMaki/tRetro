# Wave 1 使用指南：桌面打磨

這一波改動聚焦在桌面端的可發現性、版面密度、鍵盤體驗。所有功能在 1100px 以上螢幕運作最佳。

## 新功能一覽

### 1. 主棋盤 + 側欄

進入任何 retro room 後，畫面右側現在有一個固定的側欄，內含：

- **Action items**（預設顯示）— 待辦事項清單，pending 數量顯示在 tab 上
- **Sprint metrics** — 點擊切換到團隊指標面板

側欄會跟隨主棋盤捲動，但本身置頂不滑出視野。Scrum Master 可以邊看卡片邊新增 action items，不再需要捲到頁面底部。

### 2. 密度切換（compact / cozy）

`RoomHeader` 多了一個密度切換按鈕（在主題切換旁邊）。

- **cozy（預設）**— 完整顯示卡片內距與底部資訊
- **compact** — 卡片更緊湊，底部資訊（票數、評論、抽屜按鈕）淡化收起，滑鼠 hover 才完整出現

偏好會記在瀏覽器，下次重整保留。

### 3. 區塊全螢幕檢視

每個區塊（Went Well / Didn't Go Well / Thanks / Deep Discussion）的標題列右側有一個全螢幕按鈕。點擊後：

- 該區塊放大到整個視窗，卡片以多欄網格排列
- 底部固定卡片新增表單，仍可繼續即時新增
- 按 `Esc` 退出

**使用情境：**
- **送出新卡片前快速確認**：把該區塊全螢幕，掃過所有卡片避免重複
- **討論時逐一聚焦**：SM 在 discuss 階段一個區塊一個區塊放大檢視

### 4. Past retros 入口

`RoomHeader` 房名旁邊有一個「Past retros」徽章連結，點擊跳到 dashboard 並自動套用 closed 篩選，方便查看過往已關閉的 retro。

### 5. 鍵盤快捷鍵

**Dashboard 上：**

| 鍵 | 動作 |
|---|---|
| `/` | 聚焦搜尋框 |
| `N` | 開新 retro |
| `?` | 顯示快捷鍵清單 |

**進入 retro room 後：**

| 鍵 | 動作 |
|---|---|
| `N` | 聚焦第一個區塊（Went Well）的卡片新增框 |
| `A` | 側欄切到 Action items |
| `M` | 側欄切到 Sprint metrics |
| `G` `H` | 跳到 past retros |
| `G` `D` | 回 dashboard |
| `?` | 顯示快捷鍵清單 |

按 `?` 任何時候都會跳出快捷鍵浮層，按 `Esc` 關閉。

> 注意：在輸入框 / textarea 內打字時，這些單鍵快捷鍵不會觸發（除了 `Esc`），讓你正常輸入文字內容。

### 6. /join 返回鍵

加入頁面（`/room/{id}/join`）左上角現在有「Back to dashboard」連結，誤點進去也能返回。

### 7. Dashboard 卡片資訊密度

每張 retro 卡片現在顯示：

- 上方：建立日期 · `active 5m ago`（最近活動時間，依卡片/評論/action 取最新）
- 中央：4 個區塊 emoji + 目前卡片數預覽（無卡片的區塊會淡化）
- 底部：人數 / 卡片總數 / action 數

格線的最小欄寬從 260px 提升到 300px，1440p+ 螢幕現在能塞進 4–5 欄。

## 鍵盤導覽改善

所有按鈕、輸入框、tab、toggle 在「鍵盤聚焦」時會顯示明顯的紫色 focus 環（mouse 點按不會出現，避免視覺雜訊）。`Tab` 鍵巡覽整個介面時不再「失蹤」。

## 主題與輸入

亮色模式下日期選擇器的原生彈窗現在會跟著主題走（之前被強制變黑色）。

## 不再支援

- 行動裝置版面：依使用者需求，所有 retro 都在電腦上進行，不投入行動裝置 RWD 工程

## 建議使用流程

1. **建立 retro** — 在 dashboard 按 `N` 或點 New retro
2. **取暱稱進入** — 想返回時可按左上角 Back to dashboard
3. **Brainstorm 階段** — 按 `N` 直接聚焦第一個區塊；考慮切到 compact 密度看更多卡片
4. **送出前自查重複** — 點該區塊的全螢幕鈕掃一遍
5. **Discuss 階段** — SM 可逐區塊全螢幕，一張卡一張卡帶討論
6. **建 action items** — 不必滑到底，用側欄直接建立（按 `A` 切換）
7. **關房 / 匯出** — 同舊流程，header 上的按鈕未變動

如有問題或建議，請更新 `~/.claude/plans/ui-ux-retro-purring-feather.md` 計畫檔。
